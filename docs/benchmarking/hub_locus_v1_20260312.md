comparing `PoseEstimationMode::Accurate` (Structure Tensor + Weighted LM) against
`PoseEstimationMode::Fast` (IPPE + Trust-Region LM with Huber M-Estimator).

## Analysis & Insights

### 1. Tikhonov Regularization Fixes Reprojection, Not the Rotation Tail

Adding `Σ_reg = σ_n² M⁻¹ + α·I` (α = 0.25 px²) saturates the maximum information any
single corner can contribute to `1/α = 4`, preventing the Huber estimator from zeroing
out foreshortened corners entirely. The result: reprojection RMSE drops **−3.8% to −6.4%**
across all resolutions.

However, the rotation tail at 1080p (P90 = 32.6°, P99 = 74.5°) is unchanged. This means
the failure mode for the worst grazing-angle cases is upstream of the LM solver — likely a
degenerate IPPE initialization that lands in an incorrect rotation basin — not insufficient
information during refinement.

### 2. Tikhonov Has a Close-Range Cost

At 480p (frontal tags, well-conditioned corners), Trans P90 regresses from 0.0216 to
0.0255 m (+18%). This is the expected price: α = 0.25 suppresses the structure tensor's
sharpest gradients even when they are reliable, effectively throwing away valid information.
The α value should be treated as a tuning parameter: larger α → more robustness at
grazing angles, less precision at close range. `α = 0.25` is a conservative starting point.

### 3. Modes Now Essentially Tied on Reprojection

With Tikhonov applied, Accurate mode reprojection RMSE (2.30 / 3.57 / 7.48) is now
within noise of Fast mode (2.30 / 3.56 / 7.53). The regularization costs Accurate mode
its previous −4% to −6% reprojection disadvantage — it traded repro accuracy for
translation-tail robustness at 1080p.

### 4. Accurate Mode Retains P99 Translation Lead at 1080p

Despite Tikhonov, Accurate mode P99 at 1080p (0.595 m) beats Fast (0.623 m) by 5%.
The anisotropic structure-tensor covariance still provides directional uncertainty
information that the isotropic Fast mode cannot recover at extreme grazing angles.

### 5. Rotation Tail is Determined by IPPE Initialization

P90/P99 rotation errors are statistically identical between all modes and across the
Tikhonov delta. The ≥30° outliers stem from IPPE disambiguation failures (selecting the
wrong of the two IPPE solutions) on ambiguous, nearly-head-on tags. Refinement cannot
escape a wrong basin. Fixing the rotation tail requires improving IPPE disambiguation
or adding a pose-validity gate.

### 6. Detection is Unaffected

Recall, precision, and Det. RMSE are identical between modes — upstream of pose estimation.

### 7. Latency

- **Accurate mode** (sequential single-test run): ~11ms (480p) / ~23ms (720p) / ~36ms (1080p).
- **Fast mode** omits structure tensor computation and image access; expected strictly lower
  per-tag latency. Reliable comparison requires `--test-threads=1` sequential profiling.

## Methodology
- **Data Source:** Hugging Face `single_tag_locus_v1_tag36h11` subsets (45–50 images each).
- **Harness:** `locus-core` regression suite (`regression_render_tag`), Erf corner refinement.
  Snapshots: `regression_render_tag__hub_*_{fast,}.snap`
- **Environment:** Linux, Release Profile (`--release`).
- **Commit:** See git log for exact solver version used.
