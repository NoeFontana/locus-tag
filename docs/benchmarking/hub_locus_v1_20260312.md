## Analysis & Insights

### 1. GWLF: A Massive Leap in Pose Accuracy
The introduction of Gradient-Weighted Line Fitting (GWLF) with bilinear sampling has transformed the pipeline's pose estimation accuracy. By fitting infinite lines to edge gradients rather than attempting to localize blurred corners directly, we have achieved:
- **~6x improvement** in rotation P50 error at 480p (0.33° vs 2.01°).
- **~10x improvement** in rotation P50 error at 1080p (0.39° vs 4.06°).
- **100% Precision** across almost all resolutions, virtually eliminating false positives that pass geometric verification.

### 2. Efficiency Gains
GWLF is not only more accurate but also faster. It avoids the non-linear optimization required by ERF during the corner refinement phase, leading to a ~20% reduction in total pipeline latency.

### 3. Scaling to 4K (2160p)

As resolution increases to 4K, latency grows non-linearly, reaching ~100ms. This is expected as the number of pixels to process quadruples from 1080p. Detection recall drops significantly (86.67%), suggesting that at 4K, the increased pixel count might be introducing more noise or that the current thresholding/segmentation parameters need tuning for ultra-high resolutions.

### 2. Fast vs Accurate mode at 4K

At 4K, Fast mode retains a slight lead in Reprojection RMSE and Translation P50, while Accurate mode maintains a slim lead in Translation P90. The performance delta between modes becomes less pronounced at higher resolutions for these metrics.

### 3. Detection is Unaffected by Pose Mode

Recall, precision, and Det. RMSE remain identical between modes — upstream of pose estimation.

## Methodology
- **Data Source:** Hugging Face `single_tag_locus_v1_tag36h11` subsets (45–50 images each).
- **Harness:** `locus-core` regression suite (`regression_render_tag`), Erf corner refinement.
  Snapshots: `regression_render_tag__hub_*_{fast,}.snap`
- **Environment:** Linux, Release Profile (`--release`).
- **Commit:** See git log for exact solver version used.
