## Analysis & Insights

### 1. Huber Upgrade Improves Both Modes
Adding Huber-on-Mahalanobis IRLS to `refine_pose_lm_weighted` improved Accurate mode
reprojection RMSE by **−8% to −9%** and Trans P90 by **−8% to −24%** vs the prior pure-L2
weighted solver. Both modes now share the same Nielsen trust-region control and Marquardt
diagonal scaling.

### 2. Modes Are Now Close — Fast Edges Out Accurate on Repro
With both using Huber, the remaining gap between Fast and Accurate is narrow (−4% to −6% in
reprojection RMSE, favouring Fast). The Fast mode Huber threshold (1.5 px, isotropic) is
slightly more aggressive than the Accurate mode Mahalanobis threshold (k=1.345 in anisotropic
space), which explains the small persistent edge on reprojection.

### 3. 1080p P99 Translation: Accurate Leads Despite Huber
At 1080p, Accurate mode P99 translation (0.562 m) is still 11% better than Fast (0.623 m),
even after both use Huber. For heavily distorted, grazing-angle tags, the anisotropic
structure-tensor covariance is genuinely informative — it tells the optimizer which direction
each corner uncertainty is oriented, something the isotropic Huber cannot recover.

### 4. Rotation Metrics: Effectively Identical Across All Modes
P50 and P90 rotation errors agree to within 1–6% across all resolution-mode combinations.
The rotation basin is determined by IPPE initialization quality, not the refinement strategy.

### 5. Detection is Unaffected
Recall, precision, and Det. RMSE are identical between modes — upstream of pose estimation.

### 6. Latency
- **Accurate mode** (sequential single-test run): ~15.5ms (480p) / ~23.6ms (720p) / ~39.3ms (1080p).
- **Fast mode** omits structure tensor computation and image access; expected strictly lower
  per-tag latency. Reliable comparison requires `--test-threads=1` sequential profiling.

## Methodology
- **Data Source:** Hugging Face `single_tag_locus_v1_tag36h11` subsets (45–50 images each).
- **Harness:** `locus-core` regression suite (`regression_render_tag`), Erf corner refinement.
  Snapshots: `regression_render_tag__hub_*_{fast,}.snap`
- **Environment:** Linux, Release Profile (`--release`).
- **Commit:** See git log for exact solver version used.
