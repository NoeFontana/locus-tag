## Analysis & Insights

### 1. Fast Mode Wins on Reprojection and Translation Tails
The Huber M-Estimator in the new Fast mode solver outperforms the weighted Accurate mode on
reprojection RMSE (−10% to −14%) and mid-percentile translation accuracy across all resolutions.
The key mechanism: Huber dynamically down-weights corners whose 2D reprojection residual exceeds
1.5 px — a stronger outlier gate than the structure-tensor weights in Accurate mode, which can
still grant significant weight to physically blurry corners when the gradient texture is uniform.

### 2. Accurate Mode Holds the Advantage in Extreme Tails (1080p P99)
At 1080p, Fast mode P99 translation error (0.623 m) is 23% worse than Accurate mode (0.508 m).
These extreme cases represent heavily distorted tags viewed at grazing angles, where the
structure tensor provides genuinely useful anisotropic uncertainty — the Huber threshold alone
is insufficient to recover them. Accurate mode remains the correct choice when P99 robustness
matters more than median performance.

### 3. Rotation Metrics are Effectively Identical
P50 and P90 rotation errors agree to within 1–7% across all resolution-mode combinations.
Both solvers converge to the same rotation basin; the margin is driven by IPPE initialization
quality, not the refinement itself.

### 4. Detection is Unaffected
Recall, precision, and Det. RMSE are identical between modes — these depend only on the
quad extraction and decoding stages, which are upstream of pose estimation.

### 5. Latency
- **Accurate mode** (sequential single-test run): ~15.5ms (480p) / ~23.6ms (720p) / ~39.3ms (1080p).
- **Fast mode** is expected to be strictly lower-latency per tag (no structure tensor computation,
  no image access in the refinement loop). Reliable latency comparison requires sequential
  single-process profiling (`--test-threads=1`).

## Methodology
- **Data Source:** Hugging Face `single_tag_locus_v1_tag36h11` subsets (45–50 images each).
- **Harness:** `locus-core` regression suite (`regression_render_tag`), Erf corner refinement.
  Snapshots: `regression_render_tag__hub_*_{fast,}.snap`
- **Environment:** Linux, Release Profile (`--release`).
- **Commit:** See git log for exact solver version used.
