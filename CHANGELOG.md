# Changelog

All notable changes to this project will be documented here. The format
loosely follows [Keep a Changelog](https://keepachangelog.com/).

## Unreleased

### Added

- **Parallel tuning + comparative-benchmarking harness** (`tools/bench/tune/`,
  Python-only, no Rust or runtime change). New CLI subcommands drive it:
  - `bench sweep` — fan a declarative search space (`tune/spaces/*.json`) for
    Locus **and** competitors (OpenCV ArUco, pupil_apriltags) across cores; each
    worker is pinned to a single thread (`RAYON_NUM_THREADS=1` / `cv2.setNumThreads(0)`)
    so N workers never oversubscribe, and the accuracy sweep is byte-identical
    regardless of worker count. A native crash in a competitor detector is
    isolated to one errored cell instead of failing the sweep.
  - `bench tune` — select the Pareto frontier over *(maximize recall, minimize
    p99 pose error)* under hard precision/latency gates, then verify latency
    **serially** (each frontier config in its own un-pinned subprocess:
    production threading for trustworthy timing, plus native-crash isolation so a
    corrupting competitor config can't take down the tune) and guard against
    regressing the render-tag tail/mean vs shipped Locus profiles. Writes
    `pareto/<library>.json`.
  - `bench compare-report` — knob-sensitivity heatmap (which lever moves which
    metric) + per-stratum tuned-Locus-vs-best-competitor deltas + HTML bundle.
  - Competitor detector wrappers are now fully parameterizable (`from_params`),
    so comparisons tune all libraries fairly. Optional Bayesian search via the
    `[tune]` extra (`pip install -e '.[tune]'`); grid/random need no extra deps.
  - Results persist as tidy `tune_results.parquet` / `tune_configs.parquet`
    reusing the existing Tier-1 `ObservationRecord` substrate and stratification.

### Changed

- **Bench metric plumbing consolidated** (behaviour-preserving, verified
  byte-identical): the detection↔GT matcher (`tools/bench/matching.py`), the
  percentile helper (`metrics.percentiles`), and the Pareto-frontier primitive
  (`tools/bench/pareto_core.py`) are now single implementations shared by the
  headline aggregator, the Tier-1 collector, and the plot layer.

### Tests

- **Numerical test hardening (tests-only, no production-code change)**: replaced
  weak/smoke assertions with real numerical invariants and pruned redundant
  tests across the Rust and Python suites. Highlights: the homography SoA test
  now checks the canonical-square→quad reprojection instead of `data[0] != 0.0`;
  `test_decimation_accuracy` gates full-res *and* decimated corner error at
  0.01 px (was an unasserted `err1` plus a loose `err2 < 1.5`) and asserts
  decimation invariance (the observable +0.5-pixel rule); the GWLF synthetic
  test gates a measured 0.2 px cross-seed bound (was 1.0 px) and gains an
  idempotence check; two `robustness/geometry.rs` no-op (`prop_assert!(true)`)
  degenerate-quad tests now assert the solvers return `None` or *finite*
  (never NaN/Inf) output, verified over thousands of pathological quads; a new
  cross-thread/run-to-run detection determinism test and an image-edge
  robustness test were added; the tautological `DetectorConfig` default/builder
  tests were replaced by a builder-isolation invariant; and the duplicated
  `test_vectorized_poses` was de-duplicated and tightened from a `tz > 0` sign
  check to the full pose (`tz = 0.80 m`, near-identity quaternion). No public
  API or runtime-behavior change.

### Changed

- **Lint-suppression modernization**: migrated production `#[allow(...)]` lint
  suppressions to `#[expect(..., reason = "...")]` (stable since Rust 1.81)
  across `locus-core`/`locus-py`, so a suppression that stops being needed now
  fails the build (`unfulfilled_lint_expectations`) instead of lingering
  silently. The migration surfaced and removed ~20 stale or redundant
  suppressions — item allows duplicating the workspace lint table, and
  `too_many_lines`/`too_many_arguments`/`similar_names` allows on functions that
  no longer trip the lint — and deleted the dead
  `apply_adaptive_threshold_fast`/`apply_adaptive_threshold_with_params`
  integral-image path. Exported-API-sensitive lints (`unused_self`,
  `struct_field_names`) intentionally stay `#[allow]` because their firing is
  Cargo-feature-dependent. Comment/doc touch-ups in `pose.rs`, `threshold.rs`,
  and `segmentation.rs` favour why-over-what. No public API or runtime-behavior
  change.

### Documentation

- **Pose-covariance Phase 4 postmortem** (`docs/engineering/pose_covariance_phase4_postmortem_2026-05-31.md`): records the empirical falsification of Phase 4 (per-corner ERF residual-MSE covariance inflation), the last covariance-calibration lever the 2026-05-21 sweep had not ruled out. Measured end-to-end on branch `feat/activate-per-corner-empirical-noise` (PR #290): KL stays at the baseline 13.93 against the §4 gate of < 0.5, unchanged even at the theoretical upper bound, because a per-corner diagonal `σ_n²` rescale cannot fix the off-diagonal structure of `(JᵀWJ)⁻¹`; the empirical column is also all-zero in production (no low-PPB tags route through ERF on the hub corpus). The ~700 lines of SIMD/FFI/SoA plumbing are not shipped (recoverable on the branch per the §6 precedent); the audit-harness knobs and empirical bound are the salvageable artifacts. `pose_covariance_followup_2026-05-22.md` §4/§5 amended with status banners. All four covariance phases are now closed.
- **EdLines SOTA follow-up postmortem** (`docs/engineering/edlines_sota_followup_postmortem_2026-05-25.md`): records two empirical falsifications of EdLines-architecture changes attempted on 2026-05-25 — a continuous arc-balance Phase-1 mode selector (replacing the static `>40 %/<16 %` axis-imbalance gate) and the design memo's Improvement #2 (Phase-5 8-DoF line parameterisation). Both reverted on the same day after `regression_render_tag` rejected them. The postmortem catalogues the failure mechanisms (decoder-rejection invisibility for the selector; variance amplification through the line-intersection Jacobian for Phase 5) and the re-attempt conditions that would have to be resolved before either bet is worth re-opening. `edlines_sota_design_2026-05-03.md` amended with status notes on F4, F6, and Improvement #2.

Tables are snippet-included from `README.md` (single source of truth) so the numbers never drift between the two surfaces.

- Centralised SO(3) → quaternion conversion through `quat_from_so3` helper, eliminating the latent `UnitQuaternion::from_matrix` Müller-iteration hang risk on near-singular rotations across `board.rs`, `pose.rs`, and the PyO3 board-pose copy.
- Pose/board LM and ChAruco saddle refinement now emit NaN-filled covariance on singular **or near-singular** Hessian (was: identity/zero). The post-`try_inverse` finiteness gate catches the near-singular case that LU's zero-pivot test misses (tiny-but-nonzero pivot inflating the covariance to ~1e15–1e30). Production and bench LM variants are kept in sync; both now bail out after 3 consecutive Cholesky failures to surface the NaN sentinel promptly. ChAruco saddles whose per-corner Σ_c is singular (downstream of the structure-tensor gate) are dropped rather than admitted with an identity info matrix; the new rejection lane is unreachable on the production Tikhonov-regularised kernel but recorded distinctly via the new `CharucoTelemetry::rejected_reasons` discriminator buffer.
  - **Python/FFI contract**: `BoardEstimator.estimate` and `CharucoRefiner.estimate` now surface the NaN sentinel as `board_cov = None` so consumers branch on `cov is None` (matching the typed `NDArray | None` annotation). The bench-internals dict (`_bench_refine_pose_lm_weighted_with_telemetry`) preserves the NaN matrix to allow calibration audits to observe singular frames directly — `numpy.linalg.inv` does NOT raise `LinAlgError` on a NaN matrix; consumers must gate on `np.isfinite(cov).all()` (done in `tools/bench/pose_cov_audit.py`).
  - **Telemetry**: `CharucoTelemetryResult` gains a `rejected_reasons: NDArray[uint8]` buffer (0 = `LowStructureTensorDet`, 1 = `SingularCovariance`); always gate `rejected_determinants` aggregations on this discriminator, since the `SingularCovariance` lane writes NaN into the determinants buffer.
  - **Regression harness**: `regression_board_hub` test now skips NaN-cov frames from `mean_board_translation_std_m` / `mean_board_rotation_std_deg` (previously `f64::NAN.max(0.0) == 0` silently coerced the sentinel to a zero std contribution) and reports them under the new `frames_singular_covariance` snapshot field.

- Bumped `pyo3` and `numpy` from `0.28` → `0.29` to pick up the fixes for RUSTSEC-2026-0176 (OOB read in `nth` / `nth_back` on `PyList` / `PyTuple` iterators) and RUSTSEC-2026-0177 (missing `Sync` bound on `PyCFunction::new_closure`). ### Fixed

- **Debug-mode stack overflow in `Detector::with_config`** (`crates/locus-core/src/{batch,detector}.rs`): the construction call chain (`Detector::with_config` → `DetectorBuilder::build` → `FrameContext::new` → `DetectionBatch::new`) used to hold a ~215 KB `DetectionBatch` value on every stack frame because debug rustc does not perform RVO/NRVO. The original `*Box::new(Self { ... })` "workaround" in `DetectionBatch::new` did not help — the literal is still materialized on the stack, then moved to the heap, then moved back to the stack by the outer dereference. Three or more chained copies overflowed Windows main-thread stacks (1 MB default) and Linux `std::thread::spawn` stacks (2 MB default); downstream users worked around it by spawning a thread with a bigger stack. **Fix:** `DetectionBatch::new()` is replaced by `DetectionBatch::new_boxed() -> Box<Self>` (breaking — acceptable on v0.5 RC), implemented via `Box::<Self>::new_zeroed()` + `slice::fill` for the handful of non-zero default fields (`routed_to`, and the four `bench-internals` NaN / `u8::MAX` fields). `FrameContext.batch` is now `Box<DetectionBatch>` so the batch never lives inline at any level of the construction chain. `bench_api_get_batch_cloned`, `build_synthetic_batch`, and `generate_bench_batch` (which previously returned a 215 KB struct by value) now return `Box<DetectionBatch>`. A `const _: () = assert!(...)` block next to `CandidateState` / `FunnelStatus` pins both default variants to discriminant `0` at compile time, locking the SAFETY contract the unsafe block depends on. `bench_api_get_batch_cloned` now copies **all** SoA columns (the previous body silently dropped `routed_to`, `ppb_estimate`, and the four `bench-internals` telemetry columns — a pre-existing latent bug surfaced during this fix). Three new unit tests guard the contract: `construction_tests::detector_constructs_in_constrained_stack` runs the constructor inside a 256 KB stack; `test_new_boxed_defaults_full_coverage` and `test_new_boxed_bench_internals_defaults` assert the post-construction value of every non-zero default column at both endpoints (`[0]` and `[N-1]`). `constraints.md §1` now codifies the rule "Large SoA / batch structs (≥ 64 KB) must expose only `new_boxed() -> Box<Self>`" to prevent the next 200 KB SoA struct from re-introducing the same footgun.
- **`prepare_image_view` soundness gaps (F1/F2/F3)**: closed the three latent UB / lifetime issues at the Python FFI image-ingestion boundary (`crates/locus-py/src/lib.rs`). F1 — AVX2 `sample_bilinear_v8`'s 32-bit gather on 8-bit data can no longer read past the NumPy buffer. The boundary now wraps every input in a new `FfiImageBuffer` enum: when the incoming buffer already exposes ≥3 trailing bytes per row (`has_simd_padding()`), the FFI takes a zero-copy `Borrowed` fast path; otherwise it copies the image into an over-allocated padded scratch (`Padded`) so the SIMD gather guard bytes always exist. The copy fallback preserves the pre-PR public API for the natural `np.zeros((H, W), dtype=np.uint8)` idiom at the cost of one `H × W`-byte allocation + copy per `detect()` call on the under-padded path. (PR #287's initial PR-A design instead rejected under-padded inputs with `PyValueError`; the project owner chose the copy fallback over reject after the xfail-comment menu at `crates/locus-py/tests/test_ffi_contracts.py:115-122` was reconsidered.) F2 — negative `strides[0]` (e.g. from reverse-row views like `arr[::-1, :]`) is rejected before the unsigned cast that previously wrapped to a huge `usize` and built a multi-gigabyte slice in `std::slice::from_raw_parts`. F3 — the function signature now ties the returned `FfiImageBuffer<'py>` lifetime to the input `&'py PyReadonlyArray2<'_, u8>`, replacing the unconstrained `'a` that decoupled the borrow from the NumPy buffer; the call-site SAFETY comment cites the explicit tie. Resolves the Phase A1 follow-up listed at `docs/engineering/ffi_contracts.md` §1 (SIMD padding) and §7 — a new `TestNegativeStrideRejection` class covers the reverse-row path on both `detect()` and `detect_concurrent()`; `TestSimdPaddingGate.test_missing_padding_is_rejected` stays `xfail(strict=True)` as a TODO-style marker for any future variant that re-enables the strict gate.

Replaced by an inline structure-tensor determinant gate (`compute_structure_tensor_det`); the homography-projected saddle position is taken as final. Behaviour preserved. See `memory/project_refine_saddle_noop.md`.
- **Outlier-aware pose LM** (`pose.outlier_drop_d2_threshold`, default `0.0`): after the weighted LM converges, if any corner's reprojection d² = rᵢᵀ Σᵢ⁻¹ rᵢ exceeds the threshold *and* dominates the second-worst corner by ≥ 2×, the worst corner's info matrix is zeroed and the LM is re-run warm-started. The 3-corner pose is kept only when its aggregate d² over the kept corners is strictly lower than the 4-corner pose's — a self-rejection invariant that catches false-positive drops. Only `high_accuracy` opts in (threshold `25.0` ≡ 5σ²). Closes the only published SOTA gap on render-tag-hub (rotation p99 `0.8613° → 0.5607°`, −35 %) while preserving mean recall, mean corner RMSE, and translation p99.
- New `refine_pose_lm_weighted_with_info` — info-matrix-direct variant of `refine_pose_lm_weighted` that skips the 4 `Σ_c⁻¹` inversions for callers that already hold pre-inverted info. The existing `refine_pose_lm_weighted` becomes a thin wrapper that inverts and delegates.
- `DetectionBatch::outlier_corner_idx: [u8; MAX_CANDIDATES]` SoA column (`bench-internals`-gated) records which corner Phase D masked during outlier-aware refinement; sentinel `u8::MAX` ⇒ no drop. When `idx != u8::MAX` the stored pose covariance reflects 6 observations instead of 8. Documented in `docs/engineering/detection-batch-contract.md`.

### Performance

- `decode_batch_soa_generic`, `decode_batch_soa_with_camera_inner`, and `refine_poses_soa_with_config` now write SoA columns directly from rayon workers via `par_iter_mut().zip(...)`, eliminating a per-frame `Vec<TupleN>` heap allocation outside the workspace arena. New `bench-internals`-gated `contract_decode_hot_loop_zero_alloc` integration test (custom counting `GlobalAlloc`) locks the byte-delta to exactly zero on warm steady-state calls. `decoding_bench::bench_decoding_soa_realistic` median latency drops from 21.25 µs → 6.60 µs (−69 %) on AMD EPYC-Milan; pose-bench medians are within ±5 % of baseline (microbench noise floor).

### Fixed

- `decode_batch_soa_generic` now recomputes the per-candidate homography whenever **either** ERF corner refinement **or** rotation reorder mutates `batch.corners[i]`. Previously, a canonical-orientation (`rot == 0`) ERF-refined candidate kept the pre-refinement `h_slot`, leaving `(corners, H)` inconsistent and feeding stale projection geometry to `CharucoRefiner` saddle prediction — the same class of bug documented in `memory/project_refine_saddle_noop.md`. The contract is now codified in `docs/engineering/detection-batch-contract.md` §4 Phase C.
- `contract_decode_hot_loop_zero_alloc` test fixture now seeds candidate 0 with the synthetic tag's ground-truth corners so the convergence loop exercises **both** branches of the hot closure — the `FailedDecode` bypass (candidates 1..50) AND the `state == Valid` write-back path (rotation reorder, ERF refined-corners write-back, homography recompute). The previous fixture's quads never overlapped the tag, so the assertion covered only the bypass branch.
- `contract_decode_hot_loop_zero_alloc` warm-up replaced with a convergence loop (max 32 iterations, requires two consecutive `(0 bytes, 0 count)` deltas). Closes three review-flagged holes: silent `pin_rayon_to_one_thread` no-op leaving extra workers cold; shrinking `realloc` crediting 0 bytes while bumping the count; and lazily-initialised structures that need >8 warm-ups to stabilise.
- `contract_refine_poses_soa_zero_alloc_gwlf_route` added: forces `CornerRefinementMode::Gwlf` and seeds identity per-corner covariances so `compute_one`'s GWLF unpack (`Matrix2::new(covs_row[j*4..j*4+3])`) is actually exercised by the zero-alloc contract; the default route only covers the ERF branch.
- `par_iter_mut().zip(...)` chains in `decoder.rs` and `pose.rs` now guard the disjoint-slice contract with `debug_assert_eq!(...len(), n)` so a future off-by-one fix on any column fails loudly in debug rather than silently dropping the last candidate's pose / decode cells via rayon `Zip`'s shortest-input truncation.
- `docs/engineering/detection-batch-contract.md` §4: Phase C declared privileges now include `homographies[0..N]` writes (the post-rotation / post-ERF recompute that has been in the code path since 0.5.0 but was previously elided in the doc); Phase D declared privileges now list the four `bench-internals`-gated diagnostic columns (`outlier_corner_idx`, `pose_consistency_d2`, `pose_consistency_d2_max_corner`, `ippe_branch_d2_ratio`) and drop `status_mask` from the read set (Phase D consumes the upstream `partition(v)` invariant rather than reading the mask directly).

## [0.5.0] - 2026-05-20

### BREAKING CHANGES

Detector configuration has been unified around JSON profiles. Four
overlapping config paths have been collapsed into one; migration is
documented in [docs/migration/config-refactor.md](docs/migration/config-refactor.md).

- **Removed** `DetectorPreset` enum; pass `profile="standard" | "grid" | "high_accuracy"` as a string.
- **Removed** flat detector kwargs on `Detector.__init__` (`threshold_tile_size`, `quad_min_edge_score`, `refinement_mode`, etc.). Use `locus.DetectorConfig.from_profile(...)` and edit the nested model.
- **Removed** `Detector.standard_config()`; use `locus.DetectorConfig.from_profile("standard")`.
- **Removed** the `**kwargs` escape hatch on `Detector.__init__`. Unknown kwargs now raise `TypeError`.
- **Removed** duck-typed integer conversion for typed PyO3 enums at the FFI boundary. Pass the real enum instance, or a string variant name when editing a JSON-shaped dict.
- **Removed** Grid + 8-connectivity `UserWarning` branch. The grid profile pins 4-connectivity by construction.
- **Removed** `locus._profile` module. `DetectorConfig` lives in `locus._config` and is re-exported from the package root.
- **Removed** Rust `DetectorConfig::standard_default() / grid_default() / high_accuracy_default()`. Use `DetectorConfig::from_profile("standard" | "grid" | "high_accuracy")` or `DetectorConfig::from_profile_json(&str)`.
- **Removed** Rust `ConfigPreset` test enum. The regression harness now takes `.with_profile(&str)` / `.with_profile_json(&str)`.
- **Removed** Soft-decision decoding entirely. The `DecodeMode` enum (Rust `locus_core::config::DecodeMode` and Python `locus.DecodeMode`), the `decode_mode` field on `DetectorConfig` (Rust + Pydantic), the `decode_mode` key in shipped JSON profiles, the `with_decode_mode()` builder/setter, the `EdLines + Soft` cross-group validation rule, the `EdLinesIncompatibleWithSoftDecode` Rust error variant, the `regression_icra_forward_synthetic` test + `tests/fixtures/icra_synthetic.json` fixture, and the `soft_decoding_bench.rs` benchmark are all gone. Hard-decision Hamming decoding is now the only path.
- **Removed** `CornerRefinementMode::Edge` (the gradient-peak `fit_edge_line`-only refiner). Empirically dominated: no shipped profile referenced it, and the variant only surfaced in production via the wasted-work path on `Gwlf` quads (now fixed). Migration: switch to `Erf` (the PSF Gauss-Newton fit; Edge's gradient-peak fallback is still its internal failure path) or `None` (the EdLines-native sub-pixel corners). PyO3 enum discriminants shift to dense `{None=0, Erf=1, Gwlf=2}` — Rust enum order matches.
- **Removed** `PoseEstimationMode` (the `Fast` / `Accurate` toggle). The pose estimator now selects its cost surface automatically from per-corner covariance availability: image view (or external GWLF covariances) routes to weighted Mahalanobis LM with a 6×6 covariance output; no image and no covariances routes to unweighted Huber LM with no covariance. Fast mode was latency-neutral on `high_accuracy`, ≤ 13 % faster on `standard`, and 5–6× worse pose RMSE — a default-value artifact, not an architectural feature.
  - **Migration**: drop the `pose_estimation_mode=` argument from `locus.Detector.detect()` / `detect_concurrent()` calls and the `tools/cli.py` `--pose-mode` flag.
  - **Removed symbols**: `PoseEstimationMode` enum (Rust + Python), the `pose_estimation_mode` field on `DetectOptions` (Rust + Pydantic) and `DetectOptionsBuilder`, the `pose_mode` / `mode` parameter on `Detector::detect`, `Detector::detect_concurrent`, `LocusEngine::detect_with_context`, `LocusEngine::detect_concurrent`, `estimate_tag_pose`, `estimate_tag_pose_with_config`, `refine_poses_soa`, `refine_poses_soa_with_config`, and the bench-internals helpers (`bench_estimate_tag_pose`, `bench_estimate_both_branches`, `bench_refit_pose_drop_corner`). The `bench_pose_fast` divan bench is deleted along with the `regression_hub_fast_tag36h11_1080p` test and its `*_fast.snap` snapshot.

### Added

- Shipped JSON profiles (`crates/locus-core/profiles/*.json`) — single source of truth for Rust and Python. Embedded into Rust via `include_str!` and re-exposed to the Python wheel through the `_shipped_profile_json` FFI hook so both readers see byte-identical input.
- `schemas/profile.schema.json` — JSON Schema exported from the Pydantic model; CI diffs it against the runtime-generated schema to catch drift.
- `locus._config.DetectorConfig` — nested Pydantic model with cross-group invariant checks.
- `DetectorConfig::from_profile_json(&str)` on the Rust side for user-supplied profiles.
- `docs/engineering/profiles.md` — rationale for the refactor.
- `docs/migration/config-refactor.md` — before/after table for every removal.
- **Pose-consistency gate** (`pose.pose_consistency_fpr`, default `0.0` = disabled): final reprojection-consistency check inside `estimate_tag_pose_with_config` using Mahalanobis distance with χ²-derived critical values. Aggregate gate at χ²(2; fpr); per-corner gate at χ²(1; fpr). Rejected poses surface as `Detection.pose = None`. Only `high_accuracy` and `render_tag_hub` opt in (both at `1.0e-3`); `standard`, `grid`, and `general` profiles are byte-identical. Mahalanobis-aware IPPE branch selection swaps to the alternate branch when its aggregate `d²` is at least 2× better AND statistically defensible — gated on `fpr > 0` so disabled profiles preserve the legacy ideal-corner Euclidean comparison.

- **Adaptive PPB quad-extraction routing** (opt-in): new `quad.extraction_policy` enum on `DetectorConfig` with two variants — `Static` (default; preserves byte-identity for every shipped profile) and `AdaptivePpb { threshold, low_extraction, high_extraction, low_refinement, high_refinement }`. Validators reject degenerate (low == high) policies, thresholds outside `(1.0, 5.0)`, and the per-route EdLines + Erf incompatibility.

EdLines under distortion is rejected by the upstream guard, so the profile is only well-defined for pinhole / rectified inputs (matches `general` and `render_tag_hub`).

- `docs/engineering/max_recall_adaptive_calibration_20260426.md` — records the empirical calibration sweep that landed on `(threshold = 2.5, sharpening = on)`, the negative-result variants, the **intrinsic** source of the `~0.6 px` render-tag RMSE versus `high_accuracy`'s `~0.2 px`, and the deferred Phase B follow-ups (post-decode corner re-refinement, image pyramid, per-route sharpening) that would lift adaptive to a strict improvement.
- **Phase C.5 — post-decode corner re-refinement** (`decoder.post_decode_refinement`, default `false`): for each `Valid` candidate, fits the four outer tag edges with the shared `ErfEdgeFitter` (PSF step model), intersects adjacent edges to recover four sub-pixel corners, and re-solves the homography. Only `max_recall_adaptive` opts in; every other shipped profile is byte-identical. Drops the profile's ICRA-forward mean RMSE from `0.2871 → 0.2724` (−5.1 %) with recall preserved at `0.7380`, and improves render-tag mean RMSE by 0.3–0.7 % across the full `640 / 720 / 1080 / 2160` pyramid with p99 rotation flat (within sub-1 % noise). Phase A's GWLF / structure-tensor covariance is intentionally preserved — the line fit's Cramér-Rao bound is too tight for synthetic-PSF imagery and would over-trust the refit in the weighted pose solver. Two new SoA-contract tests (`contract_phase_c5_refit_valid_corners_disabled_is_noop`, `contract_phase_c5_refit_valid_corners_active_writes`) lock the `corners` + `homographies` write set.
- `docs/engineering/post_decode_refinement_20260426.md` — algorithm rationale (why edge-fit-then-intersect over per-corner saddle), failure-mode constants, the covariance-preservation argument, before/after numbers across all 5 datasets, and the byte-revert path for A/B comparison.
- **EdLines Phase-3 erf-step sub-pixel finder** (`quad.edlines_phase3_erf`, default `Disabled`): opt-in replacement for the 3-point parabolic vertex inside the EdLines Phase-3 micro-ray refinement. Fits a PSF-blurred step model `I(k) = (A+B)/2 + (B-A)/2 · erf((k-μ)/σ)` to the 5 intensity samples per probed pixel via separable nonlinear LS (alternating linear `(A,B)` solve and one Gauss-Newton step on `(μ,σ)`). Adapts σ per ray and removes the directional bias that the parabolic clamp introduced when the true edge offset exceeded ±1.5 px. Per-ray fallback to the parabolic vertex on low contrast (`Δ < 4` intensity units), singular GN, or μ runaway (`|μ| > 2`) preserves sample density. Under the new `Accurate` pose-mode default on `high_accuracy` + `EdLines`, the trade-off vs the parabolic default measured on the hub corpus is: trans_rmse `0.0047 → 0.0129 m` (single-tag, regression), rot_p99 `0.77 → 0.37 °` (single-tag, −52%), rot_p99 `40.57 → 1.19 °` (aprilgrid_golden multi-tag, −97%), multi-tag pose-flips `32 → 5` (−84%). Latency cost is `+3 %` at 1 tag/frame and `+24 %` at ~29 tags/frame, gated by `max_iters = 6` per probe. **Tail-stability vs accuracy trade-off** — no shipped profile flips the default; opt in per-deployment when rot p99 / pose-flip count is the load-bearing metric. JSON / Python accepts boolean `true`/`false` with a deprecation warning; canonical form is `"Enabled"` / `"Disabled"`.
- 5 new unit tests on `fit_erf_step_lm` (centred edge, sub-pixel offset sweep, low-contrast rejection, robustness to initial guess, variable σ).
